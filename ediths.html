базовые

Шаги Edith, отвечающие за начало/конец действия, переход между действиями, смену позы, перемещение и т.п.:

Edith Действие
Alert Вывод сообщения над человеком (мысли, слова и т.п.)
Cry Нытье (восполнение потребности, для которой невозможны действия)
Die Умереть
Finish Конец действия
GoToHuman Пойти к человеку с ID: Х
GoToObj Пойти к объекту типа Х
GoToRandomObj Пойти к рандом объекту типа Х
GoToRoom Пойти в комнату с ID: Х
GoToSystemWith Пойти в систему, где есть объект типа Х
NextAction Переход на следующее действие
Null Пустой шаг
PoseChange Смена позы
RandomChance Рандомный шанс исхода
SillyInvestigation Исследовать рандомный объект и сломать его
Start Начало действия
StopHuman Остановить человека с ID: X
Wait Подождать Х секунд

Аргументы Описание

AddRole

С v.0.16 объединена с GoToSystemWith
нет Добавляет человеку роль потребителя в системе systemActID.

Всегда успешный исход, кроме случая systemActID = -1

AddToInventory

нет Передает в инвентарь человека предмет мебели action.furn_id.


Условие успеха:
Предмет удалось передать

Условия неудачи:
action.furn_id = -1

AddToSysInventory

нет Передает в инвентарь системе systemActID предмет мебели action.furn_id.


Условие успеха:
Предмет удалось передать

Условия неудачи:
action.furn_id = -1
systemActID = -1

AgeLessThan

возраст (возрастная отметка) Проверяет, младше ли партнер из action.partner_id

Успешно, когда партнер младше и action.partner_id > -1

AgeMoreThan

возраст (возрастная отметка) Проверяет, старше ли партнер из action.partner_id

Успешно, когда партнер старше и action.partner_id > -1

Alert

текст для отображения Отображает около человека текст, в основном используется для обозначения речи. Текст каждый раз затирается после Edith.Finish

Всегда успешный исход

AskForHelp

нет Проверяет, привязана ли к запущенному действию, какая-то роль и если да, то ищет систему, которая поможет избавиться от этой роли

Условие успеха:
  • Удалось найти систему и избавиться в ней от роли (работник на месте + смог оказать помощь за выделенное время)

Условия неудачи:
  • Нет систем, где могут помочь избавиться от роли
  • Некому помочь в этой системе в данный момент времени
  • Работник этой системы не подошел на помощь за час внутриигрового времени

Buy

Deprecated since v.0.17
  1. тип мебели
  2. вид мебели
  3. кол-во мебели
  4. параметр человека р1
  5. параметр мебели р2 (опционально, может быть 0)
  6. параметр мебели р3 (опционально, может быть 0)
  7. множитель m (опционально)
например, 304 1 3 budget price markup 0.95 		Производит покупку предмета мебели человеком из другого предмета (например, со стеллажа) системы systemActID, берет оплату из параметра р1 покупателя и переводит оплату в параметр р1 человека, который является 0-м винтом системы systemActID.
Также предмет переходит в собственность человека, выполнившего этот шаг (или 0-го адреса, если нет своего).
Если вместо р2 и р3 указаны 0, то цена автоматически берется из глобальных цен всей цивилизации. Если цены нет в цивилизации, то цена автоматически устанавливается 1, множитель m тоже применяется.

Формула оплаты:
p1 -= m*p2*p3

Также у предметов мебели переопределяется свойство property, если у покупателя есть свой адрес; иначе property затирается.

Условия неудачи:
  • Не нашлось предмета (стеллажа и т.п.) с искомым предметом в достаточном количестве
  • systemActID = -1
  • action.furn_id = -1
  • Нет запрашиваемых параметров p1, p2, p3 у мебели или человека

ChangeObjType

тип мебели, вид мебели
например, 0 1		 Меняет объект мебели action.furn_id на объект с указанным типом и видом

Условие неудачи:
action.furn_id = -1

CheckDiscourse

объект мнения, тип мнения
например, soc is_useful		 Проверяет мнение по заданной теме. На вход первым параметром помимо обычных имен может принимать шорткаты:
  • soc – объектом мнения будет система systemAct
  • f – объектом мнения будет предмет мебели action.furn_id
  • a – объектом мнения будет выполняемое в данный момент действие

Условие успеха:
Мнение оказалось true или такого мнения нет

Условие неудачи:
Запрашиваемое мнение оказалось falsе

CommFulfill

параметр Проверяет, есть ли система, в этикетах которой (действиях потребителя) есть действия, восполняющие параметр и направляет следующим действием любой из этикетов любой из систем, где удовлетворяется этот параметр

Условие успеха:
  • Удалось найти систему, этикеты которой восполняют параметр
  • При успешном исходе действие всегда сразу завершается, происходит переход на этикеты найденной системы

Условие неудачи:
  • Не удалось найти систему, этикеты которой восполняют параметр

Consume

нет Употребить (уничтожить) объект action.furn_id
Условие неудачи:
action.furn_id == -1

ConsumeContent

тип контента (int) Употребить артефакт с выбранным типом контента. Занимает по 40 минут игрового времени на потребление каждой из мыслей, заложенных в артефакте.

Условие успеха:
  • Автору артефакта удалось убедить потребителя, перенято 50%+ мнений из артефакта

Условие неудачи:
  • Автору артефакта не удалось убедить потребителя, перенято менее 50% мнений из артефакта
  • Нет артефактов заданного типа в цивилизации

Cry

С 16 марта 2025 не восстанавливает потребности
время_в_сек (float), например "5.0" Используется, если невозможно выполнить действие (Action), когда например, нет нужного предмета мебели и т.п. Должно быть добавлено в Action вручную. Человек какое-то время "ноет" и восполняет потребность, с которой был связан Action на минимально возможный уровень чуть больше критического, чтобы не циклиться на этой потребности некоторое время

Всегда успешный исход

Всегда успешный исход

Условие неудачи:
