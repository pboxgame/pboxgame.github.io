Каждое действие состоит из шагов. Шаги называются Edith (в честь такой же системы из The Sims 1).

Структура действия, Action:

class Action:
   var ediths = []
   var needs = []
   var furn = []
   var id = 0
   var status = ""
ediths Список шагов, которые содержит это действие (см. далее)
needs Список параметров, при критическом уровне которых человек обратится к этому действию
furn Список типов мебели, которые нужны для выполнения этого действия
id ID этого действия в игровом мире
status Название действия, которое будет видно игроку (например, "упражняется на тренажере" или "читает книгу")

Структура Edith:

class Edith:
   var id = 0
   var true_id = 0
   var false_id = 0
   var rule = Rule.Null
   var rule_value = "0"
id Порядковый номер этого конкретного шага
true_id Номер следующего шага, если rule выполнится успешно
false_id Номер следующего шага, если rule выполнится не успешно
rule Действие, которое нужно выполнить человеку на этом конкретном шаге
rule_value Аргументы для выполнения действия (требуются не всегда)

Возможные действия (Rule) в рамках Edith:

Аргументы Описание

AddRole

sys, номер системы, номер винта, например

sys 2 0		 Добавляет человеку новую роль. Если роль временна (например, покупатель и т.д.), стоит указать -1 в качестве третьего аргумента

Всегда успешный исход

AddToInventory

тип_мебели (int), вид_мебели (int), параметры (перечисление формата "key value", опционально)

например, "201 1 seeds 100 money 500" 		Передает в инвентарь человеку предмет мебели. К предмету мебели можно опционально придать параметры перечислением, в примере слева создастся мешок кофейных зерен, в котором 100 зерен и который стоит 500 денег.
  • Если человек получил предмет в свое свободное время, то предмет перейдет к нему в инвентарь.
  • Если человек в этот момент работал и был винтом системы, предмет перейдет в инвентарь системы.


Всегда успешный исход

Alert

текст для отображения Отображает около человека текст, в основном используется для обозначения речи. Текст каждый раз затирается после Edith.Finish

Всегда успешный исход

Buy

  1. тип мебели
  2. вид мебели
  3. кол-во мебели
  4. параметр человека р1
  5. параметр мебели р2
  6. параметр мебели р3
  7. множитель m (опционально)
например, 304 1 3 budget price markup 0.95		 Производит покупку предмета мебели человеком из другого предмета (например, со стеллажа), берет оплату из параметра р1 покупателя и переводит оплату в параметр р1 человека, который является 0-м винтом системы systemAct.

Формула оплаты:
p1 -= m*p2*p3

Условия успеха (важны все сразу):
  • В предмете, из которого берется покупка (например стеллаж), есть искомый покупаемый предмет и его количества достаточно для покупки
  • У стеллажа есть маркер property с номером системы, которой он принадлежит или systemAct у действия не равен -1

Условия неудачи:
  • Не нашлось предмета (стеллажа и т.п.) с искомым предметом в достаточном количестве
  • Невозможно установить, какой системе нужно переводить деньги за оплату

ChangeObjType

тип мебели, вид мебели
например, 0 1		 Меняет объект мебели action.furn_id на объект с указанным типом и видом

Всегда успешный исход

CheckDiscourse

объект мнения, тип мнения
например, soc is_useful		 Проверяет мнение по заданной теме. На вход первым параметром помимо обычных имен может принимать шорткаты:
  • soc – объектом мнения будет система systemAct
  • f – объектом мнения будет предмет мебели action.furn_id
  • a – объектом мнения будет выполняемое в данный момент действие

Условие успеха:
Мнение оказалось true или такого мнения нет

Условие неудачи:
Запрашиваемое мнение оказалось falsе

Cry

время_в_сек (float), например "5.0" Используется, если невозможно выполнить действие (Action), когда например, нет нужного предмета мебели и т.п. Должно быть добавлено в Action вручную. Человек какое-то время "ноет" и восполняет потребность, с которой был связан Action на минимально возможный уровень чуть больше критического, чтобы не циклиться на этой потребности некоторое время

Всегда успешный исход

ExtractInfo

local_ID_человека (int) Перенять от собеседника новое мнение

Всегда успешный исход

Finish

нет Конец действия. В каждом действии является самым последним шагом.

Всегда успешный исход, в качестве true_id и false_id указывается номер этого же шага

FurnInnerChange

тип мебели, вид мебели, кол-во мебели

например, 203 1 2		 Меняет количество вложенных предметов мебели у предмета мебели action.furn_id

Успешно всегда, кроме двух условий неудачи:
  • action.furn_id == -1
  • При вычитании предметов кол-во вложенных получается меньше нуля

FurnInnerCheck

тип мебели, вид мебели, кол-во мебели

например, 203 1 2		 Проверяет количество вложенных предметов мебели у предмета мебели action.furn_id

Успешно всегда, кроме условия неудачи:
Недостаточно вложенных предметов данного типа

FurnInnerGrab

тип мебели, вид мебели, кол-во мебели

например, 203 1 2		 Взять из объекта action.furn_id заданное количество вложенных объектов данного типа. Если человек в данный момент находится в статусе винта системы, то полученные объекты уходят на баланс системы, иначе в инвентарь человека.

Успешно всегда, кроме условия неудачи:
Недостаточно вложенных предметов данного типа

FurnInnerGrabAll

Deprecated or unstable.
тип мебели, вид мебели

например, 203 1		 Взять из объекта action.furn_id все вложенные объекты данного типа. Если человек в данный момент находится в статусе винта системы, то полученные объекты уходят на баланс системы, иначе в инвентарь человека.

Всегда успешный исход

FurnInnerPut

тип мебели [...]

например, 203 204 205		 Hа вход подаются типы мебели, которые надо сгрузить в этот предмет.

Успешно всегда, кроме условия неудачи:
action.furn_id == -1

FurnInnerPutAll

тип мебели, кол-во мебели [...]

например, 203 2 204 1 205 5		 Hа вход подаются типы и количество мебели (К - З), которые надо сгрузить в этот предмет

Успешно всегда, кроме условия неудачи:

FurnParamChange

название_параметра (string), уровень_параметра (int)

например, "seeds 200" 		Изменяет значение параметра у предмета мебели, который используется в рамках этого действия.
Предмет закрепляется к действию, как правило, после прохождения шага GoToObj

Успешный исход если удалось прибавить/убавить параметр у предмета или у предмета нет запрашиваемого параметра

Условие неудачи:
  • При вычитании параметра получилось отрицательное значение

название_параметра (string), уровень_параметра (int)

например, "seeds 200" 		Проверяет параметр у предмета мебели action.furn_id.
Всегда успешно, кроме случая, когда параметр больше, чем искомый уровень

название_параметра (string), уровень_параметра (int)

ThreeParamChangeTwoFurn

Возможно будет заменен
p1 (int), p2 (int), p3 (int), m (float)

  • p1 - основной изменяемый параметр
  • р2 - множитель, параметр мебели
  • p3 - множитель, параметр мебели
  • m - коэффициент (float), на который умножается инкремент
Изменяет параметр человека, используя параметры объекта action.furn_id
Формула изменения параметра человека:
p1 += m*p2*p3

Всегда успешный исход, кроме случая, когда action.furn_id == -1

  • m - коэффициент (float), на который умножается инкремент
Инкрементирует параметр человека, используя другой параметр человека и множитель
Формула изменения параметра человека:
p1 += p2*m

Всегда успешный исход
